# Prophet vs Party 项目开发路线图与任务树 (完整版)

## 项目总体目标
构建一个**中立、教育性**的交互式平台，帮助用户系统对比共产党宣传话术与基督教、伊斯兰教传教话术的修辞技巧、心理机制和动员策略。

## 详细任务树

### 阶段 0: 项目准备（1-2天）
- [ ] 明确项目范围和敏感内容边界
- [ ] 确定最终技术栈（React + FastAPI 推荐）
- [ ] 初始化 Git 仓库和开发环境
- [ ] 创建详细时间表和里程碑

### 阶段 1: 原型迭代（最高优先级，建议1周完成）
- [ ] **完善 index_test.html**
  - 增加各类型案例（共产党10+、基督教10+、伊斯兰教10+）
  - UI 美化（响应式、暗黑模式动画）
- [ ] 设计标准案例数据结构（JSON）
- [ ] 增强可视化（词云、时间轴、相似度）

### 阶段 2: 核心功能开发（2-4周）
- [ ] 前端升级为 React / Vue3 + TypeScript
- [ ] 动态多案例对比引擎
- [ ] 案例库（搜索、过滤、分类、标签）

### 阶段 3: AI 智能增强（3-5周）
- [ ] 集成大模型（文本分析、自动报告生成）
- [ ] 修辞技巧自动识别
- [ ] 相似度算法与推荐
- [ ] "混合话术"实验功能

### 阶段 4: 后端与部署
- [ ] FastAPI 后端 + API 设计
- [ ] PostgreSQL + 向量数据库
- [ ] 认证与数据持久化
- [ ] 部署（Vercel + Railway / 阿里云）

### 阶段 5: 扩展与上线
- [ ] 多语言支持（中英）
- [ ] 完整测试与优化
- [ ] 上线发布 + 反馈收集

**MVP 最小可用产品目标**：
1. 完善原型 + 大量案例
2. React 重构
3. 基础对比功能
4. 成功部署上线
