# Prophet vs Party 项目开发路线图（优化版 v2.0）

**项目名称**：Prophet vs Party  
**项目定位**：中立、教育性交互式平台，帮助用户系统对比共产党宣传话术与基督教、伊斯兰教传教话术的修辞技巧、心理机制和动员策略。  
**核心原则**：中立性、教育性、透明性、隐私优先、避免价值判断。  
**目标用户**：学生、研究者、媒体素养爱好者、对修辞分析感兴趣的普通人。

---

## 项目总体目标

1. **MVP 目标**（2-3 周内上线）：可交互原型 + 50+ 平衡案例 + Streamlit 分析引擎 + 在线部署。
2. **中期目标**（1-2 个月）：完整案例库 + 基础 NLP 相似度分析 + React 前端 + 用户反馈机制。
3. **长期目标**：多语言支持、社区贡献、学术级数据标注、可持续部署。

### 阶段 2：Streamlit MVP 核心开发（合并原 Phase 2/3，2 周）
**目标**：将原型升级为功能完整的 Streamlit 应用，实现 NLP 基础分析。

**任务列表**：
- [ ] **Streamlit 应用开发**
  - 迁移/重构 HTML 原型到 Streamlit 多页面 App
  - 案例库页面（搜索、过滤、详情、对比）
  - 互动分析器页面（文本输入 + 自动检测 + 相似度计算）
- [ ] **NLP 基础功能**
  - 关键词 + 规则引擎（8 大话术维度）
  - 相似度计算（使用 sentence-transformers 或简单 embedding）
  - 自动报告生成（检测到的技巧 + 平行映射 + 来源标注）
  - “混合话术”实验功能（用户输入后推荐混合案例）
- [ ] **可视化增强**
  - 词云、雷达图、相似度热力图（Plotly / Altair）
  - 时间轴（历史话术演变）
- [ ] **用户交互与反馈**
  - 用户贡献案例表单（本地或简单后端存储）
  - “报告偏差 / 建议改进” 按钮 + 反馈收集
- [ ] **隐私与性能**
  - 所有分析本地运行（不上传用户输入）
  - 优化加载速度

**里程碑**：第 3 周结束时，Streamlit MVP 可通过 Streamlit Cloud / Hugging Face Spaces 在线使用。

## 技术栈最终推荐

| 阶段       | 推荐技术栈                          | 说明 |
|------------|-------------------------------------|------|
| MVP (1-3周) | Streamlit (Python) + HTML/JS 原型   | 快速开发、NLP 强、部署简单 |
| 中期       | React + TypeScript + FastAPI        | 现代前端 + 后端 API |
| 数据存储   | PostgreSQL + 本地向量（Chroma）     | 后期可升级云向量数据库 |
| 部署       | Vercel + Railway / Hugging Face     | 免费额度友好、易维护 |
| 可视化     | Plotly / Altair / Recharts          | Streamlit 原生支持 |
| NLP        | sentence-transformers + 规则引擎    | 先规则，后模型 |

---

## 风险与缓解措施

- **内容敏感性**：全程中立定位 + 来源透明 + 用户可报告偏差。
- **时间风险**：严格 MVP 优先，功能分阶段上线。
- **数据偏差**：1:1:1 比例 + 多来源 + 用户贡献机制。
- **隐私**：本地分析优先，不收集敏感输入。
- **维护**：文档完善 + 开源（推荐 GitHub 公开）。

---

## 成功指标（KPI）

- **MVP 上线后**：
  - 案例数量 ≥ 60 条（平衡比例）
  - 用户可完成一次完整分析 + 生成报告
  - 在线可用性 ≥ 99%
- **1 个月后**：
  - 用户反馈 ≥ 20 条
  - 案例贡献 ≥ 10 条（经审核）
  - 页面访问量稳定增长
